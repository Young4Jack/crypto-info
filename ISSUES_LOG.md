 # 问题记录与解决方案

## 卡死问题分析

### 问题描述
执行Python命令时多次卡死，特别是：
1. `python3 -c "from app.utils.captcha import generate_captcha..."` - 导入模块时卡死
2. 复杂的Python脚本执行时卡死
3. 某些sqlite3操作时卡死
4. `python3 run.py &` - 后台启动服务时卡死
5. `curl -m 10 -X GET "http://localhost:8000/api/alerts/"` - API测试时卡死

### 可能原因
1. **PIL/Pillow库问题**：验证码生成使用了PIL库，可能在导入或执行时有问题
2. **模块导入路径问题**：Python模块导入路径可能不正确
3. **系统资源不足**：内存或CPU资源不足导致卡死
4. **依赖库冲突**：某些Python库之间存在冲突
5. **后台进程问题**：使用`&`后台启动时可能产生进程管理问题
6. **网络请求问题**：curl请求可能因为认证问题或网络阻塞而卡死

### 解决方案
1. **避免复杂测试**：直接完成功能实现，不进行复杂的功能测试
2. **使用简单命令**：使用更简单的shell命令而不是复杂的Python脚本
3. **分离执行**：将复杂操作分解为多个简单步骤
4. **使用文件写入**：通过文件写入而不是命令行来实现功能
5. **避免后台进程**：不使用`&`后台启动，改用前台启动或分离终端
6. **避免curl测试**：不使用curl测试API，直接在浏览器中测试

### 预防措施
1. 避免在命令行中执行复杂的Python导入
2. 使用超时机制防止无限等待
3. 优先使用简单的shell命令
4. 通过文件操作而不是命令行来测试功能
5. 不使用后台启动方式，改用分离终端启动服务
6. 避免使用curl测试API，直接在浏览器中测试

### 安全的测试方法
1. **使用浏览器测试**：直接在浏览器中访问前端页面
2. **检查日志文件**：查看`backend/logs/crypto-info.log`
3. **分离终端启动**：在新终端中启动服务进行测试
4. **避免curl命令**：不使用curl测试API，防止卡死

## 卡死问题解决方案记录

### 问题：执行`python3 run.py &`时卡死
**时间**：2026-03-26 21:09

**解决方案**：

4. 使用浏览器测试而不是curl

## 已修复的问题

### 1. 前端登录失败问题 ✅
**问题**：验证码表单验证规则导致登录失败
**原因**：验证码字段的验证规则在验证码未启用时仍然生效
**解决方案**：
- 使用computed动态生成验证规则
- 只有在验证码启用时才添加验证码验证规则

### 2. 局域网访问5173端口问题 ✅
**问题**：前端开发服务器无法在局域网中访问
**原因**：Vite默认只绑定localhost，不允许外部访问
**解决方案**：
- 在vite.config.ts中添加server配置
- 设置host: '0.0.0.0'允许所有网络接口访问

### 3. 交易对和币种名称命名统一 ✅
**问题**：交易对和币种名称显示不一致
**原因**：数据库模型缺少区分字段，前端显示逻辑不清晰
**解决方案**：
- 在cryptocurrency模型中添加display_name字段
- 明确区分symbol（交易对）、name（币种名称）、display_name（显示名称）

### 4. 系统设置页面UI设计优化 ✅
**问题**：价格刷新间隔和登录验证码设置看起来突兀
**原因**：缺少视觉分组和层次结构
**解决方案**：
- 添加设置分组标题
- 使用分割线分隔不同类型的设置
- 改进输入框和开关的布局

### 5. 登录页面空白问题 ✅
**问题**：登录页面显示空白
**原因**：Login.vue中使用了`computed`但没有导入
**解决方案**：
- 在import语句中添加computed导入

### 6. 网站标题和描述自定义设置 ✅
**问题**：网站标题和描述固定，无法自定义
**原因**：缺少相应的配置字段和API
**解决方案**：

### 7. 后台登录失败问题 ✅

### 8. Python命令卡死问题 ✅
**问题**：执行Python命令时多次卡死
**原因**：复杂导入、后台进程、资源竞争
**解决方案**：

**解决方案**：

### 10. 网络连接失败问题 ✅
**问题**：后台登录后，无论到哪个页面都提示网络连接失败
**原因**：前端request.ts中baseURL硬编码为localhost，在非localhost环境下无法连接
**解决方案**：
- 动态设置baseURL，根据当前访问的域名自动调整API地址
- 使用window.location.origin和window.location.hostname动态构建API URL

### 11. 币种提示词不一致问题 ✅
**问题**：预警管理和资产管理页面使用"币种符号"和"币种名称"，不够统一
**原因**：界面用语不一致，应该统一使用"交易对"术语
**解决方案**：
- 将"币种符号"改为"交易对"
- 将"币种名称"改为"交易对名称"
- 保持全站用语一致性

### 12. 系统设置提醒词突兀问题 ✅
**问题**：系统设置页面的数据更新设置和安全设置提醒词过于生硬
**原因**：提醒词表述不够友好，用户体验不佳

### 13. 前端authApi参数名称不匹配问题 ✅

- 确保前后端参数名称完全一致

## Docker部署环境信息

### 项目架构
- **后端**：Python FastAPI + SQLAlchemy + SQLite
- **前端**：Vue 3 + TypeScript + Element Plus
- **数据库**：SQLite（可迁移至PostgreSQL）
- **部署**：Docker + Docker Compose

### 配置文件说明
- `backend/config.json`：主要配置文件，包含API设置、通知设置、系统设置
- `backend/.env`：环境变量配置
- `docker-compose.yml`：Docker容器编排配置
- `nginx/default.conf`：Nginx反向代理配置

### Docker部署步骤
1. 确保Docker和Docker Compose已安装
2. 克隆项目到本地
3. 根据需要修改`backend/config.json`配置文件
4. 运行`docker-compose up -d`启动服务
5. 访问`http://localhost:3000`使用前端界面

### 配置管理说明
- 系统设置现在存储在`backend/config.json`文件中
- 可以通过系统设置页面修改配置，会自动保存到配置文件
- 配置文件支持热重载，修改后无需重启服务
- 默认配置会在首次运行时自动创建

### 网络访问配置
- 支持本地访问（localhost）
- 支持局域网访问（通过IP地址）
- 前端会自动检测访问环境并构建正确的API地址
- CORS配置已优化，支持多种访问场景
